# v1.6.0 Release Notes

**Release Date**: 2026-04-08

---

## Summary

v1.6.0 主要聚焦于 **悬浮按钮交互体验的全面优化**，解决了多种场景下的按钮闪烁问题，并引入了多项 UI/UX 改进。

---

## Changelog

### 🎯 悬浮按钮防闪烁优化（核心改进）

#### 1. 隐藏延迟调整
- 将鼠标离开图片/按钮后的隐藏延迟从 **100ms → 500ms (0.5s)**
- 给用户更多反应时间，减少误触隐藏
- 长延迟模式 (`ihs_long_hide_delay`) 保持 1500ms 不变
- 涉及文件: `content.js` — `handleMouseLeave()` + 按钮事件委托 mouseleave 两处

#### 2. 多路径按钮区域快速移动防闪烁
- **问题**: 在多路径工具栏的多个按钮之间快速上下移动时，每个子元素的 `mouseleave` 都会独立触发隐藏定时器，导致竞态条件引起闪烁
- **修复**: 在按钮区域的 `mouseleave` 事件中增加 `relatedTarget` 检查 —— 判断鼠标离开当前子元素后进入的目标是否仍在按钮/工具栏内部。若是则忽略此次 leave 事件
- 涉及文件: `content.js` — 按钮事件委托 mouseleave 监听器

#### 3. 离开图片后快速返回防闪烁
- **问题**: 鼠标离开图片 0.5s 内移回时，之前启动的隐藏定时器仍在倒计时，到期会隐藏按钮，紧接着展示定时器又显示按钮 → 闪烁
- **修复**: 新增全局变量 `hideTimer` 追踪隐藏定时器 ID，在 `handleMouseEnter()` 中同时清除展示定时器和隐藏定时器
- 涉及文件: `content.js` — 新增 `hideTimer` 变量、修改 `handleMouseEnter()`

#### 4. 孤儿定时器防护
- **问题**: 多次快速进出时，新的隐藏定时器会覆盖旧的定时器引用，导致旧定时器变成"孤儿"——失去引用但仍在运行，在用户停住后意外触发隐藏
- **修复**: 在两处创建新 `hideTimer` 前都先 `clearTimeout(hideTimer)` 清除旧定时器
- 涉及文件: `content.js` — `handleMouseLeave()` + 按钮事件委托 mouseleave

#### 5. 图片区域实时悬停检测（同步标志位方案）
- **问题**: 隐藏定时器回调中依赖 CSS `:hover` 伪类检查，该伪类由浏览器异步更新，存在时序不确定性，导致停在图片非按钮区域时仍可能闪一下
- **修复**: 
  - 新增全局变量 `isMouseOverImage = false`，与已有的 `isMouseOverButton` 对称
  - `handleMouseEnter()` 同步设为 `true`
  - `handleMouseLeave()` 同步设为 `false`
  - 隐藏回调中用两个同步布尔标志位替代不可靠的 `:hover` 检查
- 涉及文件: `content.js`

#### 6. showDownloadButton 销毁重建优化（最终根因修复）
- **问题（最终根因）**: `showDownloadButton()` 每次调用都会先调用 `hideDownloadButton()` 销毁旧按钮再创建新按钮。当按钮已存在且是同一张图片时（如 hoverTimer 1.5s 展示定时器到期），这个销毁→创建过程导致视觉闪烁
- **修复**: `showDownloadButton()` 开头增加短路检查 —— 若 `downloadButton` 已存在且 `currentImage === img`，只执行重新定位（`positionButton`）并直接 return，跳过销毁重建流程
- 涉及文件: `content.js` — `showDownloadButton()` 函数

### 🔧 UI 改进

#### 7. 工具栏按钮间距调整
- 多路径工具栏内按钮之间的垂直间距从 **3px → 5px (+2px)**
- 涉及文件: `content.css` — `.ihs-toolbar-btn { margin-bottom }`

## 技术细节

### 防闪烁机制完整架构

```
┌─────────────────────────────────────────────────────┐
│                  事件层（同步标志位）                   │
│                                                     │
│  isMouseOverImage ←── handleMouseEnter(设true)      │
│                     └── handleMouseLeave(设false)    │
│                                                     │
│  isMouseOverButton ←── 按钮 mouseenter(设true)       │
│                      └── 按钮 mouseleave(设false)     │
│                                                     │
├─────────────────────────────────────────────────────┤
│                  定时器层                             │
│                                                     │
│  hoverTimer (1.5s) ──→ showDownloadButton()          │
│                     ├── 短路: 已有同图按钮? 只 reposition │
│                     └── 否则: 销毁+重建              │
│                                                     │
│  hideTimer (500ms) ──→ 隐藏回调                      │
│                     ├── Guard1: isMouseOverButton?   │
│                     ├── Guard2: isMouseOverImage?    │
│                     └── 通过: hideDownloadButton()    │
│                                                     │
│  定时器清理策略:                                      │
│  ├── enter 时清除 hoverTimer + hideTimer             │
│  ├── leave 时清除 hoverTimer + 旧 hideTimer           │
│  └── 按钮间移动: relatedTarget 内部跳转不创建定时器    │
│                                                     │
├─────────────────────────────────────────────────────┤
│                  DOM 层                              │
│                                                     │
│  showDownloadButton(img):                            │
│  ├── downloadButton && currentImage===img?           │
│  │   └── positionButton() + return (无 DOM 操作)      │
│  └── 否则: hideDownloadButton() + 创建新按钮         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 修改文件清单

| 文件 | 变更类型 | 说明 |
|---|---|---|
| `extension/content.js` | 核心修改 | 防闪烁全套机制：isMouseOverImage/hideTimer/短路检查/relatedTarget/同步标志位 |
| `extension/content.css` | 样式调整 | 工具栏按钮间距 +2px |
| `extension/manifest.json` | 版本升级 | 1.5.0 → 1.6.0 |
