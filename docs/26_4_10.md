# 缩略图直链下载功能

## 功能概述

按站点配置 URL 转换策略，在缩略图上悬停即可下载原图。通过正则表达式匹配缩略图 URL 并替换为原始图 URL，无需手动查找原图链接。

## 使用场景

浏览网页时，图片通常以缩略图形式展示（如 Twitter 的 `?format=jpg&name=900x900`），悬浮下载时拿到的是压缩后的缩略图而非原始分辨率。本功能允许为不同网站配置 URL 转换规则，自动将缩略图 URL 转换为原图 URL。

## 数据结构

策略存储在 `chrome.storage.sync`，key 为 `ih_url_strategies`：

```javascript
{
  id: "twitter-x-orig",
  name: "Twitter/X 原图",
  domainPattern: "twimg.com",
  enabled: true,
  isPreset: true,
  rules: [
    {
      match: "^(https?://pbs\\.twimg\\.com/media/[^?]+)\\?format=(\\w+)&name=.*$",
      replace: "$1.$2:orig"
    }
  ]
}
```

## 转换流程

```
element.src（缩略图 URL）
    ↓ findMatchingStrategy() 同时匹配页面域名和图片 URL 域名
    ↓ transformUrl() 正则替换
转换后 URL（原图 URL）
    ↓ fetch() 请求原图
    ├─ 浏览器缓存命中 → 0 流量，直接拿 blob
    └─ 缓存未命中 → 发起网络请求下载原图
    ↓ chrome.downloads.download() 保存文件
```

### 流量说明

- 缩略图 URL 和原图 URL 是不同的 HTTP 资源，转换后必定重新请求
- 若原图 URL 曾被浏览器加载过，fetch 命中缓存，不产生实际网络流量
- 若原图 URL 从未加载过，则正常发起网络请求下载

## 策略匹配规则

- 同时匹配**页面域名**（如 `x.com`）和**图片 URL 域名**（如 `pbs.twimg.com`）
- 域名匹配支持后缀匹配：`domainPattern: "twimg.com"` 可匹配 `pbs.twimg.com`
- 遍历策略数组，返回第一个 `enabled` 且域名匹配的策略
- 同域名策略互斥：启用一个策略时，自动禁用同 `domainPattern` 的其他策略

## 内置预设策略

| 预设 | 域名 | 转换规则 | 默认状态 |
|------|------|---------|---------|
| Twitter/X 原图 | twimg.com | `?format=jpg&name=xxx` → `.jpg:orig` | 启用 |
| Twitter/X 大图 | twimg.com | `?format=jpg&name=xxx` → `.jpg:large` | 禁用 |
| Reddit 原图 | redd.it | 去除 preview URL 查询参数 | 启用 |
| Imgur 直链 | imgur.com | 去除缩略图后缀 (s/m/l) | 禁用 |
| Instagram CDN | cdninstagram.com | 去除调整大小参数 | 禁用 |

## 管理页面功能

- **内置预设**：启用/禁用，不可删除
- **自定义策略**：添加/删除（名称、域名、正则匹配、替换字符串）
- **测试面板**：输入 URL 实时预览转换结果
- **导入/导出**：JSON 格式分享规则
- **规则示例**：常见正则模式参考
