# Wallhaven 原图下载（实验性功能）

## 概述

为缩略图直链下载系统新增 Wallhaven 原图下载支持。由于 Wallhaven 的页面结构与常规图片站点差异较大（叠加层遮挡、缩略图无法推导原图扩展名），采用专属 resolver 机制替代通用正则引擎，实现从页面 DOM 元数据直接构造原图 URL。

## 架构变更

### 双轨策略系统

原有策略系统仅支持正则引擎（`rules[]`），本次引入 resolver 模式，形成双轨架构：

```
策略有 rules[]    → 正则引擎转换 URL（Twitter/Reddit/Imgur/自定义）
策略有 resolver   → 专属函数从 DOM 构造 URL（Wallhaven 实验性）
```

`downloadElement` 中的策略分支：

```js
if (strategy.resolver && strategyResolvers[strategy.resolver]) {
    // 实验性：resolver 从 DOM 元数据构造 URL
    elementUrl = strategyResolvers[strategy.resolver](element);
} else {
    // 通用：正则引擎转换 URL
    const result = transformUrl(elementUrl, strategy);
    elementUrl = result.url;
}
```

### Wallhaven Resolver 工作原理

从页面 DOM 直接读取元数据，零正则、零网络探测：

```
<figure data-wallpaper-id="yqgge7">
  <img src="th.wallhaven.cc/orig/yq/yqgge7.jpg">
  <span class="png"><span>PNG</span></span>
</figure>

resolver 读取:
  data-wallpaper-id → "yqgge7"
  span.png         → "png"
  ID 前两位         → "yq"

直接构造:
  https://w.wallhaven.cc/full/yq/wallhaven-yqgge7.png
```

### 叠加层穿透检测（实验性）

Wallhaven 搜索页的 `<a class="preview">` 覆盖在 `<img>` 上方，导致鼠标事件被拦截。叠加层穿透检测仅在有匹配策略时激活，不影响无策略的网站：

```js
// 仅当候选 IMG 的 URL 匹配到已启用策略时才启用
const matchedStrategy = findMatchingStrategy(hostname, candidateImg.src);
if (matchedStrategy) {
    // 检查视觉重叠 + 排除导航链接
    element = candidateImg;
}
```

## 修改文件

| 文件 | 改动 |
|------|------|
| `extension/content.js` | 新增 `strategyResolvers` 对象和 Wallhaven resolver；`downloadElement` 增加 resolver 分支；`handleMouseEnter/Leave` 增加叠加层穿透检测（策略匹配时激活）；`BUILTIN_PRESET_STRATEGIES` 含 Wallhaven 预设；`mergeStrategiesWithPresets` 确保新预设自动补充 |
| `extension/strategies.js` | Wallhaven 预设改为 `resolver: 'wallhaven'`（无 `rules`）；`createStrategyCard` 支持 `experimental` 标记和 resolver 无 rules 的情况 |
| `extension/strategies.css` | 新增 `.experimental-tag`、`.experimental-note`、`.experimental-badge` 样式 |

## 修复的 Bug

| 问题 | 修复 |
|------|------|
| 多路径下载悬浮图标横向排布 | `showDownloadButton` 中 toolbar 改用 `display: block` |

## 策略管理页面展示

Wallhaven 策略在管理页面显示为实验性，隐藏原始正则规则，用说明文案描述工作逻辑：

- 名称旁橙色「实验性」标签
- 规则区域替换为说明文案：启用后在 Wallhaven 搜索页穿透叠加层检测图片，根据页面格式标识自动选择原图 URL
- 保留启用/禁用开关，与其他策略一致

## 验证方法

1. Chrome 加载未打包扩展
2. 访问 `https://wallhaven.cc/search` 搜索页面
3. 悬停缩略图 → 出现悬浮下载图标
4. 点击下载 → 确认文件为原图分辨率（非缩略图）
5. 分别测试 PNG 和 JPG 壁纸，确认两种格式正确识别
6. 打开策略管理页 → 确认 Wallhaven 策略显示「实验性」标记
7. 禁用 Wallhaven 策略 → 确认悬浮图标不再出现（叠加层探测不激活）
